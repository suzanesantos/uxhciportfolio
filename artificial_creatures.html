Thought Experiment

This is a thought experiment with the Homeostasis model being applied. To conduct this experiment an order automatic ordering machine at McDonalds will be used.


The needs of this device is the interaction with customers, awareness about its existence from the customers and, high orders from the customers. In adition it has the need of energy.


With motion sensors installed on it, the device will sense the customer approximating and will iniciate an conversation with the customer. From that it will start the interaction. Another external stimuli is the time with the natural light. If it is morning, for instance, at 7 am, the decive will display breakfast items and emit sound of a cup of coffee being poured.


When the customer is placing the order it can “feel” if the value of the order has a low value, as it need high orders, it will instigue the customer to add more items to the cart. When it identifies a high order value, the decide emitters cheerful sounds. When the customer takes too long to choose another item, it emits an snore sound and the screen dims out, as it “feels” sleepy. Suddenly it would turns the lights on and emmit the sound “haha I’m joking”. That would trigger the customer to be quicker if the restaurant is full.

    Exercise 8

    During this exercise, it was require to control the light emitted by the LED pin through an n LDR (Light Dependent Resistor). This resistor registered around 1021 amount of light. When covered, it registers around 1016, registering the lowest 1014. This based on the ideal circumstances. By changing the code, and adding the maximum brightness of the LED pin to the code, by mapping the sensor value.


    Arduino Code:

    float sensorValue = 0; // variable for sensor value

int greenLedPin = 10;
int yellowLedPin = 9;

int sensorValueUp = 0;
int sensorValueDown = 0;

 pinMode(sensorPin, INPUT);
 pinMode(greenLedPin, OUTPUT);

 Serial.begin(9600);
}

 sensorValue = analogRead(sensorPin); // read the value/voltage on the sensor pin and

sensorValueDown = map((1023-sensorValue), 0 , 1023, 0, 255);

    Exercise 9 C

    The goal of this exercise was to create a siren. A siren has a frequency that continuously rises until a determined upper frequency is hit. After the highest frequency is reached, the frequency declines once more. By using a loop to repeat the difference of frequencies a siren is created. Here, the lowest frequency used is 300 hertz and the highest 800 hertz. Also, 0,01 second is the delay duration of each frequency.


    Arduino Code:

    int i = 0;

void setup() {
 pinMode(8, OUTPUT);

}

void loop() {
  for(i=300;i<800;i++){
  tone(8,i);
  delay(10);
  }
  for(i=800;i>300;i--){
  tone(8,i);
  delay(10);
  }
  }



    Exercise 11 C

    During this exercise, weused an Ultrasonic Proximity Sensor, which measures distance. It basically send a pulse of ultrasonic sound. We humans can not hear it because the frequency is too high. When it returns as an echo, the sensor measures the distance based on the time of return. For the exercise it was determined that the LED pin would blink when the sensor would measures an objec within 50 centimeters.


    Arduino Code:

      const int echoPin = 6;
  const int trigPin = 7;
  int ledPin = 8;

  void setup() {
  pinMode(echoPin, INPUT);
  pinMode(trigPin, OUTPUT);
  pinMode(ledPin, OUTPUT);
  Serial.begin(9600);
  }
  void loop() {
  // send a pulse
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(5);
  digitalWrite(trigPin, LOW);
  // wait for the echo
  long duration = pulseIn(echoPin, HIGH);
  // convert the time into a distance, the speed of sound is 29 microseconds per centimeter,
  //the pulse traveled forth and back, so we divided the distance by 2
  int cm = duration / 29 / 2;

  Serial.print(cm);

  //if the measured distance is lower than 50 cm, turn on LED
  if(cm<50){
  digitalWrite(ledPin, HIGH);
  }
  //else turn off LED
  else{
   digitalWrite(ledPin, LOW);
  }
  delay(100);
  }



    Exercise 12 D

    For this exercise, both Arduino and Processing were used. Processing is a programming integrated development environment employed to prototype interactive experiences. The input from of the potentiometer exercise was applied to be transferred from Arduino to a Processing sketch. By turning to the rotatory knob clockwise or counter clockwise, the rectangle figure can move to the left or right as the sensor value is set to the x value for the position.


    Arduino Code:

        float sensorValue = 0; // variable for sensor value
    int sensorPin = A0; // variable for sensor pin
    int greenLedPin = 10; // variable for green LED pin

    void setup() {
     Serial.begin(9600); // Start the Serial connection at a
     // speed of 9600 bps
     pinMode(sensorPin, INPUT); // Input pin for potmeter or LDR
     pinMode(greenLedPin, OUTPUT); // Output pin for LED
    }

    void loop() {
     sensorValue = analogRead(sensorPin); // Read the value/current on the sensor pin and
     // store that value in the variable sensorValue
     sensorValue = (sensorValue/1023)*255; // Rescale the sensor's value. 

     // Change the values 1023 and 255 to calibratethe sensor.
     analogWrite(greenLedPin, sensorValue);// Send power to LED
     Serial.println(sensorValue); // Print the sensorValue to the serial
     // connection
     delay(100); // Wait 0.1 seconds
    }



    Exercise 13 B

    For this exercise, it was requested to invert the effect of the provided code. When uploaded the code, the LED pin would turn the light up when the button is pressed. The task was to invert this effect. The original code had LOW voltage digitalWrite for the LedPin when unpressed, and HIGH voltage digitalWrite when the pushButton is pressed. To get the opposite result, the function was changed by putting the opposite voltage value. The circuit kept unchanged.


    Arduino Code:

        // constants won't change. They're used here to set pin numbers:
    const int buttonPin = 2;     // the number of the pushbutton pin
    const int ledPin =  13;      // the number of the LED pin

    // variables will change:
    int buttonState = 0;         // variable for reading the pushbutton status

    void setup() {
      // initialize the LED pin as an output:
      pinMode(ledPin, OUTPUT);
      // initialize the pushbutton pin as an input:
      pinMode(buttonPin, INPUT);
    }

    void loop() {
      // read the state of the pushbutton value:
      buttonState = digitalRead(buttonPin);

      // check if the pushbutton is pressed. If it is, the buttonState is HIGH:
      if (buttonState == HIGH) {
        // turn LED on:
        digitalWrite(ledPin, HIGH);
      } else {
        // turn LED off:
        digitalWrite(ledPin, LOW);
      }
    }
