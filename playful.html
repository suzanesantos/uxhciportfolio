Assignment A: Concept

In order to promote more physical activity, THUAS would install a light setting on the staircase. The staircase would brights up with different color lights and intensity. Without any interaction of the students, the stairs would display a subtle light movement pattern. The light movement pattern is to trigger students to use the stairs when they pass by them. It is simple to be experienced by several users at the same time, so the play mode would be single user or cooperative.


The students use the stairs normally. Therefore, no other action from them would be required The only difference is how the step base responds to the speed of the steps.


This shows the step base would react to the students. The faster they would climb the staircase, the faster the light movement pattern would be.


Each step base would have motion sensors along the staircase. In addition to the motion sensor, an ultrasonic distance sensor dim out gradually as the user moves to the next step. That would create a sense of sequence.


LED Lamps would be placed around the step base.



Assignment B: Research

Nebula is an interactive wall that displays light in several ways. It was developed by the dutch company NYOYN in partnership with healthcare professionals Zorggroep. Nebula is addressed for adults, and mainly kids. The purpose of the wall is to trigger children to be physically challenged to do exercise, during the therapy procedures.


The goal is to make this experience a funny and engaging one. The games comprised into the wall were co-designed with physiotherapists and occupational therapists. For a more open market, the wall presents educational, social, competitive gaming, and motor activities.


The interactive wall has a capacitive touch sensor, as input technology. The continuous re-imaging of the touch profile is processed between the touch wall layer and the controller.


The output technology used is led lamps which emit light patterns and speakers to emit sound.


As the interactive wall works mostly with activities cards, through the card reader the interaction will be structured. Based on the product video, it is possible to conclude that some unstructured interaction can occur if the system is kept in standby mode through non-disturbing light patterns. This feature could have been set to often trigger users to interact with it without being too intrusive.


Users can experience it alone or with more people, at the same time. When being used with multiple users, this interaction can be competitive or cooperative, this will be also defined by the type of activity contained in the card. This shared space can be therefore used to act independently.


See more here: http://www.nyoyn.com/en/products/

    Exercise 5 E

    For this task I combined two machine learning model codes, faceapi and handpose. Faceapi will detect some face landmarks, such as mouth, nose and jaw, by outlining them. Handpose, on the other, will detect palm and track hand-skeleton, by placing 21 3D keypoints on the hand.


    For the faceapi, I changed the color of the eyes, mouth (besides its stroke weight) of the original code, and added a jaw outline to my extended code. To make it a bit more interessant, I added the handpose model to experience both models.


    Find the original code here:https://editor.p5js.org/ml5/sketches/FaceApi_Video_Landmarks

function drawBox(detections){
    for(let i = 0; i < detections.length; i++){
        const alignedRect = detections[i].alignedRect;
        const x = alignedRect._box._x
        const y = alignedRect._box._y
        const boxWidth = alignedRect._box._width
        const boxHeight  = alignedRect._box._height
        
        noFill();
        stroke(161, 95, 255);
        strokeWeight(2);
        rect(x, y, boxWidth, boxHeight);
    }
    
}
function drawLandmarks(detections){
    noFill();
    stroke(161, 95, 251)
    strokeWeight(2)
    for(let i = 0; i < detections.length; i++){
    
      const nose = detections[i].parts.nose;
                
        const rightEyeBrow = detections[i].parts.rightEyeBrow;
              
        
        const leftEyeBrow = detections[i].parts.leftEyeBrow;
        
        drawPart(nose, false);
       
        drawPart(leftEyeBrow, false);
        
        drawPart(rightEyeBrow, false);
    }
      stroke(255, 0, 0)
    strokeWeight(3)

        const mouth = detections[i].parts.mouth; 
        
        drawPart(mouth, true);
}
  stroke(200, 200,200)
    strokeWeight(2)
